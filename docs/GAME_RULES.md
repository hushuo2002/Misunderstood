# 游戏规则

## 当前推荐规则

`星际误解局` 是双向文化翻译卡牌游戏：

- 地球组回答月球文化卡。
- 月球组回答地球文化卡。
- 玩家可以选择阵营。
- 未被玩家控制的一方由 AI 代替。

## 开局

1. 选择卡组。
2. 阅读世界观导入。
3. 选择玩家阵营：Earth 或 Moon。
4. 选择胜利分数：3/5/7/9 或 1-20 自定义。
5. 随机决定先手。
6. 进入 loading。
7. 抽取初始卡牌并开始对局。

## 卡组

当前卡组：

- 综合档案 `all`
- 手势 `gesture`
- 文化 `culture`
- 习俗 `custom`
- 饮食 `food`
- 运动 `sports`
- 政治 `politics`

`all` 表示混合全部卡组。

## 回合

每个回合包含：

1. 当前队伍获得一张对方文化卡。
2. 如果是玩家队伍，玩家选择答案。
3. 如果是 AI 队伍，AI 在思考延迟后选择答案。
4. 系统判定对错。
5. 正确则当前队伍 +1 分。
6. 展示解释和深层含义。
7. 玩家点击下一轮。
8. 系统检查是否结算，否则切换队伍。

## 答题

每张卡固定 4 个选项。

```txt
selectedOption === answerIndex -> correct
```

答对：

- 当前队伍 +1 分。
- 触发成功反馈。

答错：

- 不加分。
- 触发误解反馈。

## AI

当前代码中 AI 是随机选择 4 个选项之一，理论正确率约 25%。

推荐后续改成配置化：

```ts
aiAccuracy = {
  easy: 0.65,
  normal: 0.55,
  hard: 0.45,
}
```

或者按卡牌 difficulty 变化：

```txt
difficulty 1 -> 65%
difficulty 2 -> 50%
difficulty 3 -> 35%
```

## 胜负

当前推荐默认模式：

- 任一方达到 `winScore` 后结算。
- 如果牌库先耗尽，也结算。
- 分数高者胜。
- 分数相同则平局。

未来可以拆成两种模式：

### Score Race

先到指定分数获胜。适合短局和黑客松演示。

### Archive Run

抽完牌库后结算。适合完整内容体验。

## 牌库数量

当前代码使用 `CARDS_PER_SIDE = 12` 作为剩余牌数量展示。  
正式规则建议改成实际可用卡池长度：

```txt
moonCardsRemaining = availableMoonCards.length
earthCardsRemaining = availableEarthCards.length
```

这样新增或减少卡牌时 UI 不会和真实卡池不一致。

## 设计重点

游戏不应该惩罚玩家“不知道”。  
每次错误都应该变成理解入口：

- 你为什么会这样猜？
- 月球/地球环境为什么让正确答案成立？
- 这个习俗背后的风险、资源或社会结构是什么？
